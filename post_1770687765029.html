🤖 AI 코딩, 10%만 쓰고 계신가요? 서브에이전트로 생산성 10배 높이는 법

"이제는 개발자가 아니라 CEO처럼 생각하세요"
AI 시대의 새로운 개발 패러다임


안녕하세요~

오늘은 클로드 코드를 제대로 활용하는 방법에 대해 이야기해보려고 해요.
혹시 지금 AI한테 그냥 채팅으로만 코드 짜달라고 하고 계신가요?
그렇다면 AI 기능의 10%밖에 못 쓰고 계신 거예요!



💡 구시대적 개발 방식에서 벗어나세요

많은 분들이 이렇게 개발하고 계세요.
설계는 내가 직접 하고, 데이터베이스 디자인도 내가 짜고,
AI한테는 코드 구현만 시키고, 테스트는 또 내가 직접 하고...

이건 정말 비효율적인 방식입니다.

혹시 이런 경험 있으신가요?
"왜 코드 결과물이 매번 달라지지?"
"같은 요청을 했는데 또 다른 구조로 만들어주네?"

이게 바로 AI를 제대로 컨트롤하지 못해서 생기는 문제예요.
AI가 하는 일이 블랙박스가 되어버리는 거죠.





✌️ CEO 마인드로 개발하세요

앞으로는 CEO처럼 생각하셔야 해요.

회사 CEO가 디테일하게 요구사항을 말하나요?
아니죠. 그냥 "올해는 중고래 플랫폼 만들어봐" 이렇게 간단하게 얘기하잖아요.

그럼 기획자가 기획하고, 개발자가 개발하고,
QA 담당자가 테스트하고, 이 사이클이 돌아가죠.

이 모든 직원을 AI로 만드는 게 핵심입니다.

여러분은 사장님으로서 "뭘 만들고 싶다"만 간단히 말하면,
서브에이전트들이 알아서 계획 짜고, 개발하고, 테스트까지 다 해주는 거예요.





🎯 서브에이전트 만드는 법

서브에이전트는 생각보다 쉽게 만들 수 있어요.

1단계: 에이전트 생성
클로드 코드에서 슬래시(/) 입력 → 'Create New Agent' 클릭
"요구사항을 구체화하고 계획을 짜주는 계획 에이전트를 만들어줘" 이렇게만 입력하면 끝!

2단계: 역할 부여
- 계획 에이전트: 요구사항 정리, 데이터베이스 설계, 폴더 구조 설계
- 구현 에이전트: 시니어 개발자 관점에서 확장성 좋은 클린 아키텍처 코드 작성
- 테스트 에이전트: 계획 문서 대비 실제 구현 분석 및 누락 체크

처음엔 드래프트 버전이지만,
프로젝트 하면서 계속 수정하고 키워나가면 돼요.
신입사원 키우듯이요!





📋 스킬스(Skills)로 규칙 정하기

에이전트를 만들었다면, 이제 회사 규칙을 만들어야 해요.

스킬스는 모든 에이전트가 따라야 하는 공통 규칙이에요.
신입사원 온보딩 문서 같은 거죠.

예를 들면:
- "비전공자도 이해할 수 있게 코드마다 한국어 주석 달아줘"
- "리액트 컴포넌트는 함수형으로만 작성"
- "에러 처리는 try-catch 필수"

이런 규칙들을 `.md` 파일로 만들어두면,
모든 에이전트가 이 규칙을 따라서 일관성 있는 코드를 만들어줘요.

꿀팁: 스킬스 마켓플레이스(Agent Skills Marketplace)에 가면
다른 개발자들이 만들어둔 스킬 파일을 무료로 다운받을 수 있어요!





⚡ 커맨드로 더 편하게

"매번 '서브에이전트 이용해서 스킬스 참고해서...' 이렇게 쓰기 귀찮은데요?"

그럼 커스텀 커맨드를 만드세요!

예를 들어 `/develop`이라는 커맨드를 만들면:
→ 자동으로 서브에이전트 실행
→ 스킬스 파일 참고
→ 기능 구현까지 한 번에!

`/develop 마감일 기능 추가해줘"
이렇게 한 줄만 입력하면 모든 게 자동으로 돌아가요.





🚀 실전 예시: 비키트(VKit) 활용하기

"서브에이전트를 직접 만들기 귀찮은데..."
괜찮아요! 이미 잘 만들어진 에이전트 팩이 있거든요.

비키트(VKit)는 실제 개발 회사에서 쓰는 워크플로우를 그대로 담은 서브에이전트 모음이에요.
오픈소스라 무료로 쓸 수 있고요!

PDCA 사이클로 자동화:

Plan (계획)
→ `/pdca-plan 투두앱 만들어줘`
→ 기능 요구사항, 데이터 구조, 파일 구조, UI 레이아웃까지 문서화

Design (설계)
→ `/pdca-design`
→ 시스템 아키텍처, 함수 플로우, 코드 구조까지 상세 설계

Do (구현)
→ `/pdca-do`
→ 설계 문서 그대로 코드 생성

Check (검증)
→ `/pdca-analyze`
→ 계획 vs 실제 구현 일치도 분석 (96% 이상이면 우수!)
→ 누락된 기능 자동 체크

Act (개선)
→ `/pdca-iterate`
→ 일치도 90% 이하일 때 자동 수정

어머, 정말인가요?
네! 이 모든 게 커맨드 몇 번으로 자동화돼요.





✨ 일관성이 핵심입니다

서브에이전트의 가장 큰 장점은 결과의 일관성이에요.

예전에는:
- 같은 요청을 해도 매번 다른 구조로 코드가 나옴
- AI가 멋대로 기능 추가하거나 빼버림
- 내가 원하는 걸 정확히 표현하려고 프롬프트 엄청 길게 써야 함

서브에이전트 사용 후:
- 계획 문서가 있으니 항상 같은 구조로 생성됨
- 문서에 명시된 기능만 정확히 구현
- 프로젝트 날아가도 문서만 있으면 똑같이 복구 가능

심지어 96% 이상 일치도까지 보장해주니,
이제 AI 결과물을 신뢰할 수 있게 된 거예요!





🎓 이제 어디에 집중해야 할까요?

개발자의 에너지를 어디에 써야 할까요?

Before:
- 설계: 내가 직접 ❌
- 개발: AI에게 요청 ⭐
- 테스트: 내가 직접 ❌
- 문서화: 내가 직접 ❌

After (서브에이전트):
- 설계: 계획 에이전트 ⭐
- 개발: 구현 에이전트 ⭐
- 테스트: 분석 에이전트 ⭐
- 문서화: 자동 생성 ⭐
- 나의 역할: 에이전트 최적화 연구 💡

이제 우리는 "어떻게 더 똑똑한 AI 직원을 만들 것인가"에 집중해야 해요.
코드 짜는 시간은 줄이고, 전략 세우는 시간은 늘리는 거죠!

특히 비키트 같은 오픈소스 에이전트 팩을 보면서

여러분도 지금 당장 서브에이전트 한 번 만들어보세요.
처음엔 서툴어도 괜찮아요. 신입사원 키우듯이 천천히 발전시키면 되니까요!
