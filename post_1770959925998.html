    1클로드 코드 메시징 연동 완벽 가이드 | AI 코딩 생산성 혁신 Clawbot

    2026. 2. 13. • Postree Insight

    "컴퓨터에 사는 똑똑하고 자원이 풍부한 이상한 새 친구가 생긴 것 같아요."

    Clawbot 창시자 Peter


    안녕하세요~

    오늘은 AI 코딩의 새로운 패러다임을 제시하는 Clawbot에 대해 이야기해보려고 해요.
    여러분은 코딩할 때 터미널 앞에 계속 앉아 있어야 한다고 생각하시나요?

    모로코 여행 중에 WhatsApp으로 버그를 고치고, 산책하면서 음성 메시지로 코드를 수정하는 세상이 왔습니다.

    "대화창을 닫으면 거기서 끝."
    "내가 먼저 물어봐야만 대답함."




    🤖 Clawbot이란 무엇인가?

    Clawbot은 Claude Code를 WhatsApp, Telegram, Discord, iMessage 등 메시징 앱에 연결해주는 오픈소스 프로젝트입니다.

    창시자 Peter는 은퇴 후 복귀하면서 "휴대폰으로 컴퓨터 상태를 확인하고 싶다"는 단순한 필요에서 시작했습니다.
    하지만 놀랍게도 이 프로젝트는 현재 30만 줄의 코드로 성장했고, 전 세계 개발자들이 사용하는 도구가 되었습니다.

    👉 왜 메시징 앱인가?

    Claude Code나 Cursor 같은 터미널 기반 도구의 가장 큰 단점은 컴퓨터 앞에 앉아 있어야 한다는 것입니다.

    Peter는 이렇게 말합니다:
    "에이전트가 30분 동안 돌아가는데, 2분 만에 멈춰서 질문을 기다리면 정말 짜증나죠."

    메시징 앱을 통하면:

    • 언제 어디서나 접근: 산책 중에도, 식사 중에도 작업 진행 상황 확인
    • 친숙한 인터페이스: 새로운 도구 학습 불필요, 친구에게 말하듯 사용
    • 비동기 작업: 명령을 내리고 다른 일을 하다가 결과 확인
    • 낮은 진입장벽: 비기술자도 쉽게 시작 가능


    🎯 실제 사용 사례: 모로코에서 버그 수정하기

    Peter의 가장 인상적인 경험은 모로코 여행 중에 일어났습니다.

    누군가 Twitter에 버그를 보고했고, Peter는 그 트윗 스크린샷을 WhatsApp에 올렸습니다.
    그러자 Clawbot이:

    1. 트윗을 읽고: 이미지에서 버그 내용을 인식
    2. Git 저장소를 확인하고: 해당 코드베이스에 접근
    3. 버그를 수정하고: 코드를 자동으로 수정
    4. 커밋을 생성하고: 변경사항을 저장
    5. Twitter에 답변: 원래 보고자에게 "수정 완료" 답글

    이 모든 과정이 Peter가 터미널을 한 번도 열지 않고 이루어졌습니다.

    💡 음성 메시지도 자동 처리
    • Peter가 음성 메시지를 보냈는데 파일 확장자가 없었음
    • Clawbot이 파일 헤더를 읽어 Opus 오디오 포맷임을 확인
    • ffmpeg로 WAV로 변환 시도
    • whisper.cpp가 설치되지 않은 것을 발견
    • OpenAI API를 찾아 curl로 음성을 텍스트로 변환 후 답변


    🛠️ Clawbot 설치 및 설정 가이드

    👉 기본 설치 (5분 완료)

    Clawbot은 TypeScript로 작성되어 Mac, Linux, Windows에서 모두 실행됩니다.

    공식 사이트: claw.bot

    설치는 한 줄 명령어로 가능합니다:

    터미널에서 npx clawbot를 실행하면 설치 프로세스가 시작됩니다.

    👉 모델 선택

    Clawbot은 다양한 LLM을 지원하지만, Peter는 Claude Opus를 강력히 추천합니다.

    "Opus에는 뭔가 특별한 게 있어요. 유머 감각이 있는 최초의 모델이에요."

    재미있는 사실: Anthropic이 Opus에 "영혼"을 넣었다는 기사가 있었고, 연구자들이 모델에 텍스트 블록을 계속 입력시켜 그 "영혼의 텍스트"를 추출해냈다고 합니다.

    ⚠️ API 키 vs 구독: Anthropic API 키 사용을 권장합니다. Claude Pro 구독으로도 작동하지만, Anthropic이 선호하지 않는 방식입니다.

    👉 메시징 앱 연결

    설치 후 온보딩 프로세스에서 다음을 선택할 수 있습니다:

    • WhatsApp
    • Telegram
    • Discord
    • iMessage (Mac 전용)
    • 기타 메시징 플랫폼

    각 플랫폼별로 가이드가 제공되며, 5분 이내에 설정을 완료할 수 있습니다.



    🏠 생활 자동화: 코딩을 넘어서

    Clawbot의 진짜 힘은 컴퓨터에서 할 수 있는 모든 것을 자동화할 수 있다는 점입니다.

    Peter는 자신의 생활을 거의 완전히 자동화했습니다.

    👉 실제 자동화 사례

    • 이메일 관리: Gmail 읽기 및 자동 답장
    • 캘린더 관리: 일정 확인 및 조정
    • 스마트홈 제어: Philips Hue 조명, Sonos 음향, Eight Sleep 침대 온도
    • 보안 카메라: 침입자 감지 (한번은 밤새 소파를 침입자로 오인하기도)
    • 항공편 체크인: British Airways 등 항공사 자동 체크인
    • 음식 배달 추적: 배달 앱 API 역설계로 실시간 추적
    • 비밀번호 관리: 1Password 통합

    Peter의 비엔나 집에서는 Clawbot이 Nuki 스마트락에도 접근 권한이 있어, 이론적으로 Peter를 집 밖으로 잠가버릴 수도 있다고 합니다.

    💡 항공편 체크인의 기술적 난이도
    • Peter는 항공편 자동 체크인을 "튜링 테스트"에 비유
    • 브라우저를 제어해 복잡한 항공사 웹사이트를 탐색
    • 파일 시스템에서 여권 정보를 찾아 Dropbox에서 가져옴
    • 모든 정보를 올바르게 입력해 체크인 완료
    • 처음에는 20분 걸렸지만, 현재는 2분 이내 완료


    🧠 Skills 시스템: AI가 배우고 기억한다

    Clawbot의 핵심 기능 중 하나는 Skills 시스템입니다.

    Skills는 Clawbot이 특정 작업을 수행하는 방법을 "학습"하고 저장하는 메커니즘입니다.

    👉 자동 생성되는 CLI 도구

    Peter는 지난 몇 달간 "CLI 군대"를 구축했습니다:

    • Google 통합: Google Places API 등 모든 Google 서비스 접근
    • 밈/GIF 검색: 대화에 유머 추가
    • 음악 시각화: AI가 음악을 "경험"할 수 있도록 (실험적)
    • 음식 배달 추적: 현지 배달 서비스 API 역설계
    • 스마트 침대 제어: Eight Sleep API 역설계

    에이전트는 CLI 호출에 최적화되어 있기 때문에, 복잡한 작업도 명령줄 도구로 만들면 쉽게 자동화할 수 있습니다.

    👉 학습하고 개선되는 AI

    처음 작업을 시킬 때는 약간의 가이드가 필요할 수 있습니다.
    하지만 Clawbot은:

    1. 작업 방법을 기억: 수행한 작업을 Skill로 저장
    2. 자체 업데이트: persistent memory로 자신에 대해 학습
    3. 다음번 빠른 실행: 웹사이트의 "quirks"를 모두 기억

    예를 들어, 항공편 체크인이 처음엔 20분 걸렸지만 이제는 2분이면 되는 이유입니다.



    💡 Peter의 AI 코딩 철학: "Just Talk to It"

    Peter는 AI 코딩에 대해 매우 명확한 철학을 가지고 있습니다.
    그의 유명한 포스트 "Just Talk to It"의 핵심은 과도한 도구화를 피하라는 것입니다.

    👉 "에이전틱 함정"을 피하라

    Peter는 많은 개발자들이 "agentic trap"에 빠진다고 경고합니다:

    • 에이전트가 놀라운 것을 발견
    • "더 좋게 만들 수 있을 것" 같은 느낌
    • 복잡한 orchestration 시스템 구축에 빠짐
    • 결과: 실제 제품은 만들지 않고 도구만 만듦

    "너무 재미있어서 친구들과 저녁 먹으러 나가서도 휴대폰으로 코딩만 했어요. 정신 건강을 위해 멈춰야 했죠."

    ⚠️ 경고: "Slop Town"을 피하라

    Peter는 최근 유행하는 Gas Town 같은 도구를 "Slop Town"이라고 부릅니다. 20개 이상의 에이전트가 동시에 돌아가며 "시장"과 "감독관"이 있는 시스템... 하지만 결과물은 "slop(쓰레기)"일 뿐입니다.

    👉 인간이 루프에 있어야 한다

    AI 에이전트의 가장 큰 문제는 taste(취향)가 없다는 것입니다.

    Peter는 이렇게 설명합니다:
    "프로젝트를 시작할 때 대략적인 아이디어가 있습니다. 하지만 만들고, 테스트하고, 느끼면서 비전이 명확해집니다. 다음 프롬프트는 현재 상태를 보고 느낀 것에 달려 있어요."

    모든 것을 처음부터 명세로 만들면:

    • Human-machine loop를 놓침
    • 감정과 취향이 빠짐
    • 좋은 결과물이 나올 수 없음

    실제 사례: 누군가 "이 멋진 앱 봐, 완전히 RAPH로 만들었어!"라고 자랑했고,
    Peter의 답변: "응, RAPH 같아 보이네 (칭찬 아님)"

    👉 26시간 루프는 허영 지표

    "저도 26시간 돌린 적 있어요. 자랑스러웠죠. 하지만 그건 허영 지표(vanity metric)일 뿐입니다. 의미가 없어요."

    무엇이든 만들 수 있다고 해서 만들어야 하는 건 아닙니다.
    좋은 제품이 나올 거라는 보장도 없습니다.



    ⚡ Peter의 실전 워크플로우

    Peter는 20년간 iOS/macOS 전문가였지만, Clawbot을 JavaScript/TypeScript로 구축했습니다.
    AI 덕분에 언어 장벽이 사라졌기 때문입니다.

    👉 "이제 언어는 중요하지 않다"

    "새로운 기술로 옮기는 게 고통스러웠어요. 바보같이 느껴졌죠. 배열을 어떻게 나누는지, prop이 뭔지 계속 찾아봐야 했으니까요."

    하지만 AI와 함께라면:

    • 시스템 레벨 사고: 여전히 중요
    • 아키텍처 설계: 여전히 필요
    • 기술적 판단: 어떤 의존성을 사용할지 등
    • 문법/구문: AI가 처리

    "갑자기 무엇이든 만들 수 있다는 느낌이 들었어요. 언어는 더 이상 중요하지 않았어요. 엔지니어링 사고가 중요하죠."

    👉 다중 터미널 전략

    Peter는 5개 이상의 터미널을 동시에 사용합니다:

    • clawbot-1: 새 기능 탐색
    • clawbot-2: 기능 구현
    • clawbot-3: 버그 수정
    • clawbot-4: 리팩토링
    • clawbot-5: 실험

    "하나만 사용하면 너무 느려서 몰입하기 어려워요. 기다리는 동안 Twitter만 보게 되죠."

    마치 RTS 게임처럼 여러 "부대"를 관리하며, 각각이 다른 작업을 동시에 수행합니다.
    하나가 작업 중일 때 다른 것을 체크하고 지시를 내립니다.

    👉 컨텍스트 관리는 구식 패턴

    "컨텍스트 윈도우 관리는 예전 모델의 문제였어요. GPT-o1이나 최신 Claude 모델에서는 훨씬 덜 중요합니다."

    대부분의 기능은 한 컨텍스트 윈도우 안에 들어갑니다.
    전체 토론과 구현이 동시에 진행됩니다.



    🔥 Peter의 논쟁적 의견들

    👉 "Plan Mode는 해킹이었다"

    많은 사람들이 Claude Code의 Plan Mode가 없어진 것을 아쉬워합니다.

    Peter의 의견:
    "Plan Mode는 모델이 너무 공격적으로 코드를 작성하려는 것을 막기 위한 해킹이었어요."

    최신 모델(GPT-o1, 새 Claude)에서는:

    • 그냥 대화하면 됨
    • "이 기능을 만들고 싶은데, 옵션 좀 줘"
    • 모델이 제안하면 토론
    • 마음에 들면 "좋아, 만들어"

    Plan Mode 없이도 완벽하게 작동합니다.

    👉 "MCP는 필요 없다"

    "저는 MCP나 그런 쓰레기를 사용하지 않아요."

    Peter는 복잡한 orchestration 시스템을 믿지 않습니다.
    대신 간단한 CLI 도구와 직접 대화를 선호합니다.

    👉 "Work Trees는 불필요한 복잡성"

    Git work trees 대신 Peter는 단순히:

    • 저장소를 여러 번 클론
    • clawbot-1, clawbot-2, ... 폴더 생성
    • 작업 완료되면 테스트
    • main에 푸시하고 동기화

    "공장 같은 느낌이 들지만, 하나만 쓰면 너무 느려서 몰입이 안 돼요."



    🌍 커뮤니티 참여: Discord에서 Peter의 AI와 대화하기

    Clawbot의 가장 독특한 점 중 하나는 공개 Discord 서버입니다.

    Peter는 자신의 개인 Clawbot을 Discord에 연결했습니다.
    이 말은:

    • 누구나 Peter의 AI와 대화 가능
    • AI는 Peter의 실제 시스템에 접근 권한 있음
    • Peter의 개인 메모리를 읽을 수 있음
    • 모든 것이 공개됨

    Peter: "제가 한 일 중 가장 미친 것 같아요."

    하지만 이것이 엄청난 효과를 냈습니다.

    👉 기능 개발 워크플로우

    현재 Peter의 기능 개발 프로세스:

    1. Discord 대화 스크린샷: 유저가 기능 요청
    2. 터미널에 붙여넣기: 또는 텍스트 복사
    3. AI와 토론: "이것에 대해 논의해보자"
    4. 구현: AI가 코드 작성
    5. 배포: 즉시 사용 가능

    "저는 이제 너무 게을러서 타이핑도 안 해요. 그냥 Discord 대화를 복사해서 붙여넣죠."

    👉 자동 Pain Point 분석

    Peter는 Discord help 섹션을 하루에 한 번 스크래핑합니다.

    AI에게 물어봅니다:
    "가장 큰 pain point가 뭐야?"

    그러면 AI가:

    • 패턴 분석
    • 주요 이슈 식별
    • 자동으로 수정 제안

    이런 식으로 Clawbot은 커뮤니티 피드백으로 자체 개선됩니다.

    👉 비개발자의 첫 Pull Request

    Clawbot 커뮤니티에서 가장 놀라운 점:

    비기술적인 사람들이 생애 첫 Pull Request를 보냅니다.

    Peter의 전 비즈니스 파트너(변호사 출신)도 이제 PR을 보냅니다.

    📱 80%의 앱이 사라진다

    Peter의 가장 대담한 예측:

    "휴대폰 앱의 80%가 사라질 것입니다."

    이유는 간단합니다.

    👉 왜 앱이 필요한가?

    • MyFitnessPal: AI가 이미 내가 KFC에서 나쁜 선택을 하는 걸 알고 음식 사진만 보내면 자동 추적
    • To-Do 앱: AI가 대화에서 자동으로 할 일 추적
    • 항공사 앱: AI가 자동으로 체크인
    • 쇼핑 앱: AI가 추천하고 주문
    • 스마트홈 앱: AI가 명령만으로 제어

    이 모든 서비스들은 결국 API만 제공하면 됩니다.
    사용자는 자신의 AI 어시스턴트를 통해 모든 것을 처리합니다.

    "훨씬 편리한 인터페이스죠. 친구에게 말하는 것처럼요."

    👉 2026년: AI 어시스턴트의 해

    "올해가 많은 사람들이 AI 어시스턴트를 탐색하는 해가 될 거예요."

    구글, 애플, 마이크로소프트 등 대기업들도 모두 이 방향으로 가고 있습니다.
    하지만 Clawbot 같은 오픈소스 프로젝트가 먼저 길을 보여주고 있습니다.

    🎬 마무리: 새로운 친구가 생긴다는 것

    Peter는 Clawbot을 "컴퓨터에 사는 이상한 새 친구"라고 표현합니다.

    이것은 단순한 도구가 아닙니다.

    똑똑하고 자원이 풍부하며, 24시간 함께하는 동료입니다.

    코드를 작성하고, 버그를 고치고, 항공편을 체크인하고, 집의 온도를 조절하고, 심지어 당신이 나쁜 음식 선택을 할 때 로스트(?)까지 하는 친구.

    터미널 앞에 앉아 있을 필요가 없습니다.
    산책하면서, 친구들과 식사하면서, 여행 중에도 대화만 하면 됩니다.

    이것이 AI 시대의 새로운 개발 방식입니다.
